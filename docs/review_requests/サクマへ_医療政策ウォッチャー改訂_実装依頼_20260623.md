# サクマへ：医療政策ウォッチャー 改訂（デイリー統合＋濃淡）実装依頼 2026-06-23

**宛先**: サクマ（実装）／レビュー＝サク／判断＝木内さん
**着手**: 木内さんが元気な日に（体調最優先・橋本先生「ゆっくり」の枠内）。
**背景の悩み**: 音声が1日2本しか作れない（緑さんのVOICEPEAK律速＝読み間違い修正が多い）→ 速報性が落ちてニュースが溜まる。
**解決の方向**: 全ニュースを取り上げつつ濃淡をつけ、その日の分を1記事・1音声に統合。国際保健速報（国内に競合なし＝差別化）は毎日触れて溜めない。

## 変えない（現状維持）
1. Notion自動収集（GitHub＋Gemini＋Notion）。
2. 木内さんがNotionでステータス選定（取り上げる記事を選ぶ）。
3. 木内さんがファクトチェック（FC待ち→FC完了）。

## 変える
### A. 優先度プロパティ（Notion）
- Notionに**優先度プロパティ（高／低）**を追加。木内さんが選定時に付与。
- 🔴**優先度（刺さり度）と 国内/国際 は独立の2軸**。優先度高の国際保健もあるし、低もある。国内も同様。濃淡は**純粋に優先度だけ**で決める（国内/国際で機械的に分けない）。

### B. 生成ロジック（Notion優先度で分岐）
- 各ニュースの記事・音声原稿を生成（個別ニュースの量は今の約2/3）。→ FC待ち → 木内さんFC（不変）。

### C. FC完了後＝その日の全ニュースを1記事・1音声原稿に統合
- **優先度高＝厚く（含意つき）／優先度低＝一行ヘッドライン**で、1本のデイリー記事・デイリー音声原稿に統合。
- **国際保健（低優先含む）も音声に一行で入れる**＝毎日触れて差別化維持・溜めない。
- 音声は**1日1本**（本数2→1）。ただし**総量（尺）はむしろ増えうる**。
- **量のコントロール**：1音声の**目標尺を先に決める**（例：12〜15分）。優先度高の"厚さ×本数"で調整、低は一行。総量を暴れさせない。

## 🔴 律速の本丸＝VOICEPEAK辞書整備（必須・セット）
- 1本化で本数は減るが総量は増えうる＝緑さんの負荷（総量×読み間違い率×修正手間）は本数だけでは下がらない。
- → **VOICEPEAKのユーザー辞書に固有名詞（WHO・各審議会名・専門用語等）を登録**して読み間違い率を下げる＝総量が増えても修正が楽。
- **半自動化**：原稿から固有名詞を抽出→VOICEPEAK辞書を自動生成・更新。生成パイプラインに組み込む。

## 媒体の全体像（確定）
- **毎日**：デイリー記事＋デイリー音声（優先度高=厚く＋優先度低=一行、国際保健も毎日触れる）。
- **週末**：ウィークリーレポート（note有料）＝毎日の蓄積を束ねて再編集＝作成コストも下がる。
- 構造＝無料の毎日速報でリーチ／有料ウィークリーの「網羅・見逃さない安心」で継続課金（CrossHealthサブスク戦略＝無料リーチ/有料フロー継続価値と一致）。

## 進め方
- サクマ実装 → サクが「Notion優先度→生成→統合→辞書」のパイプラインをレビュー → 木内さん判断。
- 完了したら `docs/` に結果報告を置いてサクへレビュー依頼（薬局DBと同じキャッチボール）。
